\secrel{Основания разработки}

Язык \bi\ первоначально был создан для чтения файлов конфигурации и исходных
данных для расчетных программ в удобном текстовом формате\ref{appconfigs}.

Затем возникла идея использования \bi\ в качестве \termdef{языка-шаблонизатора}:
на вход транслятора подается файл-шаблон\note{файл конфигурации или программа
на любом языке програмирования}, в шаблоне с помощью \termdef{маркеров}
\verb|$маркеры$| прописаны фрагменты кода на языке \bi\--- \termdef{макросы}.
\bi-макросы вычисляются транлятором, и их текстовое значение подставляется
вместо маркеров.

\bigskip
Перспективные направления:

\begin{itemize}[nosep]
\item развитие языка до \st-подобной интерактивной
динамической рабочей среды, с функциями автогенерации кода\note{Java/Android,
\cpp, чистый Си для встраиваемых систем}\ для заданных объектов (разработка
мультиплаформенного ПО), 
\item функционалом CAD/GUI (создание АРМ\note{автоматизированных рабчих мест})
\item богатой математической библиотекой для символьных и численных вычислений и
\item поддержкой облачных и кластерных архитектур
\end{itemize}
